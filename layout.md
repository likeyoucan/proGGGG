# Название игры

- Нужно название игры

# 1. Экраны и навигация

## 1.1. Preloader

- Дисклеймер
- Экран студии:
  - название студии
  - изображение / логотип студии
- Загрузка:
  - индикатор загрузки

## 1.2. Главное меню

- Продолжить
  - активно, если есть сохранения
- Новая игра
  - создание имени персонажа
  - дополнительные параметры, если нужны
- Загрузить
  - активно, если есть сохранения
- Достижения
  - опционально
- Авторы
- Настройки
- Выход

### Загрузка сохранения

- Список сохранений:
  - название
  - дата
  - время
- Кнопки:
  - Назад
  - Загрузить
    - активна, если выбрано сохранение

### Процесс загрузки

- Подсказки
- Индикатор
- Фоновые изображения

### Дополнительно

- Политика конфиденциальности
  - располагается в нижней части экрана

### Вибрация

- Вибрация
  - переключатель

### Графика

  - низкое
  - среднее

  - максимальное
- Яркость

- Коррекция цветов

### Контент и монетизация

- Скрыть NSFW

  - переключатель

### Навигация

- Назад

## 1.4. Экран игры

### Верхняя левая часть

- Аватар
- Имя игрока
- Энергия
- Попытки

### Верхняя правая часть

- Список вариантов ответа

### Основная сцена

- Слой 1: фон
- Слой 2: персонажи и предметы

## 1.5. Меню паузы

- Продолжить
- Новая игра
  - с предупреждением
- Сохранить
- Загрузить
- Настройки
- В главное меню
- Выход из игры

## 1.6. Инвентарь

- Список полученных предметов
- Название предмета
- Описание предмета
- Количество, если применимо
- Использование предмета в сценах или выборах

## 1.7. Карта сюжета

- Отображение текущей главы и сцены
- Просмотр пройденных узлов
- Просмотр выбранных вариантов
- Отображение связей между сценами
- Недоступные ветки могут быть скрыты или отмечены отдельно

# 2. Игровая логика

## 2.1. Структура сюжета

- Игра состоит из глав (Chapters)
  - каждая глава имеет:
    - порядковый номер
    - название

- Главы состоят из сцен / узлов (Scenes / Nodes)
  - каждая сцена имеет:
    - ID
    - название
    - контент

- Сцены связаны через варианты выбора (Choices / Edges)
  - каждый вариант имеет:
    - текст
    - ссылку на следующую сцену
    - переход внутри текущей главы или в другую главу

- Разработчик может задавать эффекты перехода между сценами
  в зависимости от выбора игрока

## 2.2. Визуальная система сцены

### Background

- Статичное изображение
- Серия изображений
- Слайдшоу с плавным переходом
- Зацикленное видео

### Midground

- Персонажи
- Объекты
- Располагается поверх фона

### Foreground

- Наложения
- Атмосферные эффекты
- Дождь
- Блики
- Прочие визуальные эффекты поверх сцены

### Поддержка качества медиа

- Для каждого медиафайла должна быть возможность хранить версии разного качества:
  - Low
  - Medium
  - High
  - Max

- Игра должна загружать нужный файл в зависимости от выбранных настроек графики

## 2.3. Игровые ресурсы

### Энергия

- Тратится на прохождение
- Если энергия закончилась, игрок не может выбрать действие
- Игроку предлагается:
  - посмотреть рекламу
  - купить Premium

### Попытки

- Ограничивают количество попыток на выбор правильного варианта ответа

### Инвентарь

- Хранит предметы, полученные по ходу сюжета

# 3. Системная логика настроек

## 3.1. Аудио

- Для звука, музыки и SFX используются:
  - тумблер Вкл / Выкл
  - ползунок громкости от 0 до 100%

### Правила

- Если тумблер выключен, ползунок устанавливается в 0
- Если ползунок установлен в 0, тумблер выключается
- Если ползунок поднят выше 0, тумблер включается
- Если тумблер включают после выключения, ползунок возвращается на предыдущее значение

## 3.2. Вибрация

- Для вибрации используются:
  - тумблер Вкл / Выкл
  - ползунок силы вибрации

### Правила

- Если тумблер выключен, ползунок устанавливается в 0
- Если ползунок установлен в 0, тумблер выключается
- Если ползунок поднят выше 0, тумблер включается
- Если тумблер включают после выключения, ползунок возвращается на предыдущее значение

# 4. Монетизация и реклама

## 4.1. Premium

- Глобальный флаг:
  - куплена
  - не куплена

- Если Premium куплена:
  - скрывается вся реклама
  - энергия становится бесконечной

- Если игрок пытается отключить рекламу в настройках без Premium:
  - вызывается окно покупки Premium

## 4.2. Interstitial Ads

- Разработчик может назначать показ полноэкранной рекламы:
  - при переходе между главами
  - при переходе между сценами
  - при выходе в главное меню

- Настройка задаётся в свойствах сцены или перехода

## 4.3. Rewarded Ads

- Игрок может смотреть рекламу за вознаграждение для:
  - восстановления энергии
  - получения подсказки
  - покупки правильного ответа
